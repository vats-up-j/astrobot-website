## Decision framework for new features

When evaluating any new feature or idea, score it on:

### 1. Traffic potential (0-5)
- Will this page/feature rank in Google for searches people actually make?
- Is the search volume daily (5), seasonal (3), or one-time (1)?

### 2. Conversion potential (0-5)
- Does this naturally lead to "get personalized reading on WhatsApp"?
- Is the user in a decision-making mindset when they use this?

### 3. Effort (0-5, inverted — 5 = easy, 0 = very hard)
- Can it be built static (no backend)?
- How many hours of work?
- Does it need ongoing maintenance?

### 4. Differentiation (0-5)
- Do competitors already do this well? (If yes, score 1)
- Does this leverage HoroTalk's unique AI + Vedic angle? (If yes, score 5)

**Score = (Traffic + Conversion + Effort + Differentiation) / 4**
- Above 3.5: Build it now
- 2.5 - 3.5: Put on roadmap
- Below 2.5: Skip or revisit later

## Review questions for every new idea

1. **Who is this for?** Sita, Karan, or someone else?
2. **What search query does this capture?** (If none, it won't drive traffic)
3. **How does this lead to WhatsApp?** (If it doesn't, it doesn't make money)
4. **Can we build it static?** (If no, it costs money to host)
5. **Will it still work in 6 months without maintenance?** (If no, reconsider)
